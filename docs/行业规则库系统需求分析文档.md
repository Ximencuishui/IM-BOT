# 行业规则库系统需求分析文档

## 1. 引言

### 1.1 文档目的
本文档旨在分析 TonjClaw 项目中不同行业的通用功能模块，设计统一的行业规则库系统架构，制定报表格式规范，并提出规则库的管理机制。通过标准化和模块化设计，实现不同行业通过导入行业特定规则来配置机器人的信息收发逻辑、业务流程和交互模式。

### 1.2 项目背景
TonjClaw 是一款本地化配送服务商智能订货统计工具，目前已支持 20+ 行业预设规则模板。随着业务扩展，需要构建更完善的规则库系统，支持多行业、多场景的灵活配置和快速部署。

### 3.2 规则分类体系

| 规则类型 | 子类型 | 用途 | 数据结构 |
|---------|-------|------|---------|
| **解析规则** | regex/keyword/custom | 将非结构化消息解析为结构化数据 | `rule_name, rule_type, pattern, extract_fields, priority` |
| **统计规则** | daily/weekly/monthly/custom | 定义数据统计维度和图表类型 | `rule_name, stat_type, dimensions, filters, chart_type` |
| **回复规则** | keyword/regex/event | 定义机器人自动回复逻辑 | `rule_name, trigger_type, trigger_content, reply_type, reply_content` |
| **流程规则** | approval/workflow/condition | 定义业务流程和审批逻辑 | `rule_name, flow_type, conditions, actions, transitions` |
| **报表规则** | template/query/style | 定义报表模板和格式规范 | `rule_name, report_type, template_data, query_config` |

### 3.3 规则库数据结构

#### 3.3.1 规则模板表 (RuleTemplate)

```sql
CREATE TABLE t_rule_template (
    id INTEGER PRIMARY KEY AUTO_INCREMENT COMMENT '模板ID',
    template_name VARCHAR(100) NOT NULL COMMENT '模板名称',
    industry VARCHAR(50) NOT NULL COMMENT '所属行业',
    category VARCHAR(50) COMMENT '分类：通用/专业/定制',
    description VARCHAR(500) COMMENT '模板描述',
    
    -- 规则内容（JSON格式）
    parse_rules TEXT COMMENT '解析规则JSON数组',
    stat_rules TEXT COMMENT '统计规则JSON数组',
    reply_rules TEXT COMMENT '回复规则JSON数组',
    flow_rules TEXT COMMENT '流程规则JSON数组',
    report_rules TEXT COMMENT '报表规则JSON数组',
    
    -- 版本信息
    version VARCHAR(20) DEFAULT '1.0' COMMENT '版本号',
    version_history TEXT COMMENT '版本历史JSON',
    
    -- 来源与归属
    source_type VARCHAR(20) DEFAULT 'official' COMMENT '来源: official/community/user',
    author_id INTEGER COMMENT '上传用户ID',
    organization_id INTEGER COMMENT '所属组织ID',
    
    -- 统计信息
    download_count INTEGER DEFAULT 0 COMMENT '下载次数',
    rating NUMERIC(3,2) DEFAULT 0.00 COMMENT '评分',
    rating_count INTEGER DEFAULT 0 COMMENT '评分人数',
    usage_count INTEGER DEFAULT 0 COMMENT '使用次数',
    
    -- 状态控制
    is_public BOOLEAN DEFAULT TRUE COMMENT '是否公开',
    is_featured BOOLEAN DEFAULT FALSE COMMENT '是否精选',
    is_active BOOLEAN DEFAULT TRUE COMMENT '是否启用',
    is_deprecated BOOLEAN DEFAULT FALSE COMMENT '是否废弃',
    
    -- 元数据
    tags VARCHAR(200) COMMENT '标签，逗号分隔',
    compatible_versions VARCHAR(200) COMMENT '兼容版本',
    dependencies TEXT COMMENT '依赖规则ID列表',
    
    -- 时间戳
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    published_at DATETIME COMMENT '发布时间',
    
    INDEX idx_industry (industry),
    INDEX idx_source_type (source_type),
    INDEX idx_is_active (is_active)
);
```

#### 3.3.2 解析规则表 (ParseRule)

```sql
CREATE TABLE t_parse_rule (
    id INTEGER PRIMARY KEY AUTO_INCREMENT COMMENT '规则ID',
    user_id INTEGER NOT NULL COMMENT '用户ID',
    template_id INTEGER COMMENT '关联模板ID',
    
    rule_name VARCHAR(100) NOT NULL COMMENT '规则名称',
    rule_type VARCHAR(20) NOT NULL DEFAULT 'regex' COMMENT '规则类型: regex/keyword/custom',
    pattern TEXT COMMENT '匹配模式',
    extract_fields TEXT COMMENT '提取字段配置JSON',
    priority INTEGER DEFAULT 0 COMMENT '优先级（数值越大越优先）',
    is_active BOOLEAN DEFAULT TRUE COMMENT '是否启用',
    
    -- 高级配置
    match_mode VARCHAR(20) DEFAULT 'exact' COMMENT '匹配模式: exact/contains/regex',
    ignore_case BOOLEAN DEFAULT TRUE COMMENT '是否忽略大小写',
    stop_on_match BOOLEAN DEFAULT TRUE COMMENT '匹配后是否停止',
    fallback_rule_id INTEGER COMMENT '匹配失败时回退规则ID',
    
    -- 执行配置
    timeout INTEGER DEFAULT 5000 COMMENT '执行超时时间(ms)',
    max_retries INTEGER DEFAULT 3 COMMENT '最大重试次数',
    
    -- 统计信息
    match_count INTEGER DEFAULT 0 COMMENT '匹配成功次数',
    fail_count INTEGER DEFAULT 0 COMMENT '匹配失败次数',
    
    description VARCHAR(500) COMMENT '规则描述',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    FOREIGN KEY (user_id) REFERENCES t_user_account(id),
    FOREIGN KEY (template_id) REFERENCES t_rule_template(id),
    INDEX idx_user_id (user_id),
    INDEX idx_rule_type (rule_type),
    INDEX idx_priority (priority)
);
```

#### 3.3.3 回复规则表 (ReplyRule)

```sql
    -- 回复配置
    reply_type VARCHAR(20) DEFAULT 'text' COMMENT '回复类型: text/image/file/link/markdown',
    reply_content TEXT COMMENT '回复内容',
    reply_template TEXT COMMENT '回复模板（支持变量替换）',
    
    -- 执行控制

    cooldown_seconds INTEGER DEFAULT 0 COMMENT '冷却时间(秒)',
    max_executions INTEGER DEFAULT -1 COMMENT '最大执行次数(-1为无限)',
    execution_count INTEGER DEFAULT 0 COMMENT '已执行次数',
    
    -- 目标配置
    target_type VARCHAR(20) DEFAULT 'same' COMMENT '回复目标: same/group/private/boss',
    target_id VARCHAR(100) COMMENT '目标ID',
    
    -- 条件判断
    conditions TEXT COMMENT '前置条件JSON',

    INDEX idx_trigger_type (trigger_type)
);
```

## 5. 规则库管理机制

### 5.1 规则版本控制

#### 5.1.1 版本号规范

```
主版本号.次版本号.修订号
示例: 1.0.0, 1.1.0, 2.0.1
```

| 版本类型 | 变更内容 | 示例 |
|---------|---------|------|
| **主版本** | 不兼容的API变更、重大功能重构 | 1.0.0 → 2.0.0 |
| **次版本** | 向后兼容的新功能 | 1.0.0 → 1.1.0 |
| **修订号** | 向后兼容的问题修复 | 1.0.0 → 1.0.1 |

#### 5.1.2 版本管理流程

```
创建新版本

    ▼

└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 版本测试     │→ 功能测试
│              │→ 性能测试
│              │→ 兼容性测试
└──────┬───────┘
       │
       ▼
┌──────────────┐

       ▼

│ 版本归档     │→ 旧版本标记为历史
│              │→ 保留回滚能力
└──────────────┘
```

### 6.4 潜在风险

| 风险类型 | 风险描述 | 影响程度 | 应对措施 |
|---------|---------|---------|---------|
| **兼容性风险** | 旧版规则与新版规则引擎不兼容 | 高 | 提供规则迁移工具和兼容模式 |
| **性能风险** | 规则数量增多导致匹配效率下降 | 中 | 实现规则索引和缓存机制 |
| **安全风险** | 用户自定义规则可能包含恶意代码 | 高 | 规则沙箱执行、格式校验、权限控制 |
| **复杂度风险** | 规则系统过于复杂，用户难以使用 | 中 | 提供可视化规则编辑器、预设模板 |
| **数据一致性** | 规则更新可能导致数据不一致 | 中 | 事务管理、版本回滚机制 |

## 9. 总结

本需求分析文档基于 TonjClaw 项目现有架构，提出了一套完整的行业规则库系统设计方案：

1. **通用模块识别**: 识别出消息解析、统计报表、自动回复等跨行业通用模块，设计三层架构（核心基础层、规则引擎层、行业应用层）

2. **规则库架构**: 扩展现有规则系统，新增流程规则和报表规则，设计完整的规则分类体系和数据结构

3. **报表规范**: 制定统一的报表格式规范，支持日报/周报/月报/自定义报表，提供 Markdown 格式输出示例

4. **管理机制**: 设计规则版本控制、更新流程和权限管理体系，确保规则库的安全、稳定和可维护

5. **可行性分析**: 基于现有技术基础，制定分阶段实施路径，识别潜在风险并提出应对措施

该方案将使 TonjClaw 能够快速适配不同行业场景，通过导入行业规则模板实现机器人的灵活配置，为后续业务扩展奠定坚实基础。

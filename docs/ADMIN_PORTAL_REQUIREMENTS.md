# Admin Portal 功能需求说明

## 📋 概述

Admin Portal 是系统的管理员后台，提供系统级的管理功能，用于管理用户、授权码、定价、规则库、推广等核心业务。

**访问路径**: `http://your-domain.com/admin-portal`  
**部署位置**: 云服务器或公网可访问的服务器  
**目标用户**: 系统管理员、运营人员

---

## 🎯 核心功能模块

### 1. 👥 用户管理

#### 功能描述
管理系统中的所有用户账户，包括创建、编辑、删除、权限设置等。

#### 详细功能
- **用户列表**
  - 查看所有注册用户
  - 搜索和筛选（按用户名、邮箱、角色、状态）
  - 分页显示
  
- **用户操作**
  - 创建新用户（手动创建或导入）
  - 编辑用户信息（用户名、邮箱、手机号、公司名）
  - 设置用户角色（超级管理员、管理员、普通用户）
  - 禁用/启用账户
  - 删除用户（软删除，保留数据）
  
- **用户详情**
  - 查看用户基本信息
  - 查看用户活跃度统计
  - 查看登录历史
  - 查看关联的授权码
  - 查看订单历史
  
- **批量操作**
  - 批量导入用户（CSV/Excel）
  - 批量导出用户数据
  - 批量禁用/启用

#### 数据字段
```
- id: 用户ID
- username: 用户名
- email: 邮箱
- phone: 手机号
- company_name: 公司名称
- role: 角色（super_admin）
- is_active: 是否激活
- subscription_type: 订阅类型（monthly/yearly）
- max_groups: 最大群组数
- created_at: 注册时间
- last_login: 最后登录时间
```

---

### 2. 🔑 授权码管理

#### 功能描述
管理系统中的所有授权码，包括生成、撤销、展期、销售统计等。

#### 详细功能
- **授权码列表**
  - 查看所有授权码
  - 搜索和筛选（按授权码、用户、状态、到期时间）
  - 分页显示
  
- **授权码生成**
  - 单个生成（指定用户、有效期）
  - 批量生成（指定数量、有效期、套餐类型）
  - 自定义授权码格式
  
- **授权码操作**
  - 查看授权码详情
  - 撤销授权码（立即失效）
  - 批量展期（为多个授权码延长有效期）
  - 手动激活/停用
  
- **授权码统计**
  - 总授权码数量
  - 活跃授权码数量
  - 已过期授权码数量
  - 即将到期预警（7天、30天）
  - 授权码销售趋势图
  
- **授权码同步**
  - 查看桌面端提交的授权验证记录
  - 机器指纹管理
  - 设备绑定管理

#### 数据字段
```
- id: 授权码ID
- license_key: 授权码
- user_id: 关联用户ID
- product_id: 关联产品ID
- status: 状态（active/expired/revoked）
- activated_at: 激活时间
- expires_at: 到期时间
- machine_fingerprint: 机器指纹
- created_at: 创建时间
```

### 3. 💰 定价配置

#### 功能描述
管理产品的定价策略，以组合套餐形式定义产品。

#### 详细功能
- **产品管理**
  - 创建新产品（组合套餐）
  - 编辑产品信息（名称、描述、包含内容）
  - 设置产品价格（月付/年付）
  - 启用/禁用产品
  - 删除产品（软删除）
  
- **套餐配置**
  - 定义套餐内容（功能模块、使用限制）
  - 设置折扣规则（早鸟价、团购价）
  - 配置试用期
  - 价格历史记录
  
- **定价策略**
  - 基础定价设置
  - 阶梯定价（按用户数、按用量）
  - 促销活动管理
  - 优惠券管理
  
- **价格展示**
  - 前台价格页面配置
  - 价格对比表
  - 推荐套餐标记

#### 数据字段
```
- id: 产品ID
- product_name: 产品名称
- description: 产品描述
- features: 功能特性（JSON）
- monthly_price: 月付价格
- yearly_price: 年付价格
- trial_days: 试用天数
- is_active: 是否启用
- sort_order: 排序
- created_at: 创建时间
```

### 4. 📚 规则库管理

#### 功能描述
管理系统的总规则库，包括解析规则、统计规则的增删改查，以及查看各用户的规则备份。

#### 详细功能
- **总规则库管理**
  - 创建新规则模板
  - 编辑规则（解析规则、统计规则）
  - 删除规则模板
  - 规则分类管理（按行业、场景）
  - 规则版本控制
  
- **规则模板分发**
  - 将规则模板推送给用户
  - 查看规则使用情况统计
  - 规则更新通知
  
- **用户规则备份查看**
  - 查看各用户备份的规则
  - 规则对比（用户规则 vs 总库规则）
  - 规则同步建议
  - 下载用户规则备份
  
- **用户授权监控**
  - 查看用户授权到期状态
  - 规则使用权限控制
  - 到期提醒设置
  
- **规则导入导出**
  - 批量导入规则（CSV/TXT/MD）
  - 导出规则库
  - 规则备份与恢复

#### 数据字段
```
RuleTemplate:
- id: 模板ID
- template_name: 模板名称
- industry: 所属行业
- description: 描述
- parse_rules: 解析规则（JSON）
- stat_rules: 统计规则（JSON）
- version: 版本号
- is_preset: 是否预设
- created_at: 创建时间
- updated_at: 更新时间

UserRuleBackup:
- id: 备份ID
- user_id: 用户ID
- rule_content: 规则内容
- backup_time: 备份时间
- license_status: 授权状态
```

### 5. 📊 订单与续费管理

#### 功能描述
监控用户到期状况，管理续费流程，分析授权码销售情况。

#### 详细功能
- **用户到期监控**
  - 查看所有用户授权到期时间
  - 即将到期预警（7天、15天、30天）
  - 已过期用户列表
  - 到期趋势图表
  
- **自动提醒付费**
  - 配置提醒规则（提前几天提醒）
  - 提醒渠道设置（邮件、站内信、短信）
  - 提醒模板管理
  - 提醒发送记录
  - 手动发送提醒
  
- **授权码销售分析**
  - 销售数据统计（日/周/月/年）
  - 产品销售排行
  - 销售渠道分析
  - 收入统计报表
  - 转化率分析
  
- **续费管理**
  - 续费历史记录
  - 续费率统计
  - 流失用户分析
  - 续费预测模型
  
- **订单管理**
  - 查看所有订单
  - 订单状态跟踪
  - 退款处理
  - 发票管理

#### 数据字段
```

- id: 记录ID
- user_id: 用户ID

- old_expiry: 原到期时间
- new_expiry: 新到期时间
- amount: 金额
- payment_method: 支付方式
- renewed_at: 续费时间

NotificationLog:
- id: 日志ID
- user_id: 用户ID
- notification_type: 通知类型
- sent_at: 发送时间
- status: 发送状态
```

## 🔐 权限控制

### 角色定义
- **超级管理员（super_admin）**: 所有权限
- **管理员（admin）**: 大部分管理权限，部分敏感操作受限
- **运营人员（operator）**: 用户管理、订单管理、推广管理
- **客服（support）**: 只读权限，查看用户和订单信息

### 权限矩阵
| 功能模块 | super_admin | admin | operator | support |
|---------|-------------|-------|----------|---------|
| 用户管理 | ✅ | ✅ | ✅ | 👁️ |
| 授权码管理 | ✅ | ✅ | ✅ | 👁️ |
| 定价配置 | ✅ | ✅ | ❌ | ❌ |
| 规则库管理 | ✅ | ✅ | ✅ | 👁️ |
| 订单与续费 | ✅ | ✅ | ✅ | 👁️ |
| 推广管理 | ✅ | ✅ | ✅ | ❌ |
| 系统监控 | ✅ | ✅ | 👁️ | ❌ |
| 审计日志 | ✅ | ✅ | 👁️ | ❌ |
| 系统设置 | ✅ | ⚠️ | ❌ | ❌ |

注：✅ 完全权限，⚠️ 部分权限，👁️ 只读，❌ 无权限

## 📝 注意事项

1. **数据隔离**: 确保不同租户/用户的数据严格隔离
2. **性能优化**: 大量数据时需要分页和缓存
3. **备份策略**: 定期备份重要数据
4. **日志保留**: 审计日志至少保留 6 个月
5. **合规性**: 遵循相关法律法规（隐私保护、数据安全）
6. **可扩展性**: 预留扩展接口，便于未来功能增加

# 平场工程队现场管理助手系统需求文档

**文档版本**: 1.0  
**日期**: 2026年6月15日  
**项目代号**: TonjClaw-Construction-Field-Assistant  
**适用行业**: 平场工程、土方施工、基础建设工程队  

---

## 1. 行业背景与市场分析

### 1.1 行业背景

平场工程是建筑施工的基础环节，主要涉及场地平整、土方开挖、回填压实等工作。平场工程队普遍面临以下痛点：

| 痛点类别 | 具体问题 |
|----------|----------|
| **管理粗放** | 工地信息分散、员工档案不规范、设备租赁记录混乱 |
| **数据滞后** | 工程量统计依赖人工上报、费用报销周期长、库存信息不实时 |
| **沟通低效** | 依赖微信群聊口头汇报、信息容易遗漏、缺乏系统化交互 |
| **决策困难** | 缺乏数据支撑、成本核算不准确、工程进度难以把控 |

### 1.2 目标市场定位

本系统定位于**中小型平场工程队**和**施工班组**，帮助其实现现场管理数字化、智能化，通过微信聊天交互方式降低使用门槛，提升管理效率和决策能力。

---

## 2. 产品定位与目标用户

### 2.1 产品定位

**平场工程队现场管理助手**：一款基于微信生态的智能化现场管理系统，通过自然语言交互方式，实现工地管理、员工管理、工程量统计、费用管理、耗材管理、设备租赁管理和记账管理的全流程数字化。

### 2.2 用户角色与权限

| 用户角色 | 权限等级 | 核心功能 |
|----------|----------|----------|
| **系统管理员（BOSS）** | 最高权限 | 所有模块的管理、审批、报表查看、系统设置 |
| **项目经理/工头** | 中级权限 | 工地管理、员工调度、工程量审核、费用审批 |
| **普通工人** | 基础权限 | 工程量上报、费用报销、耗材出入库记录 |

### 3.2 员工管理模块

#### 3.2.1 功能描述

| 功能点 | 说明 |
|--------|------|
| 员工信息管理 | 新增员工基本信息（姓名、联系方式、岗位等）及工资标准设置 |
| 员工信息维护 | 通过私聊方式进行员工信息更新、调岗、调薪等操作 |
| 薪资自动计算 | 根据入离职时间、工资标准、考勤记录自动计算工资 |
| 薪资报表生成 | 按周、按月、按季度生成薪资汇总报表 |

#### 3.2.2 核心字段定义

| 字段名称 | 类型 | 必填 | 说明 |
|----------|------|------|------|
| 员工姓名 | 字符串 | 是 | 真实姓名 |
| 联系方式 | 字符串 | 是 | 手机号码 |
| 岗位 | 枚举 | 是 | 司机/挖掘机手/铲车司机/普工/管理 |
| 工资标准 | 数值 | 是 | 日薪或月薪（元） |
| 工资类型 | 枚举 | 是 | 日结/周结/月结/计件 |
| 入职日期 | 日期 | 是 | 入职时间 |
| 离职日期 | 日期 | 否 | 离职时间（在职时为空） |
| 所属工地 | 关联 | 是 | 当前工作工地 |
| 考勤记录 | 关联 | 否 | 出勤、请假、旷工记录 |

#### 3.2.3 薪资计算规则

| 计算项 | 规则 |
|--------|------|
| 基础工资 | 工资标准 × 出勤天数 |
| 加班工资 | 加班天数 × 工资标准 × 1.5 |
| 请假扣款 | 请假天数 × 工资标准 |
| 计件工资 | 完成量 × 计件单价 |
| 应发工资 | 基础工资 + 加班工资 - 请假扣款 + 计件工资 |

| 用户故事 | 描述 |
|----------|------|
| 作为BOSS，我希望添加员工信息 | 建立员工档案 |
| 作为BOSS，我希望通过私聊调薪 | 灵活调整员工薪资 |
| 作为财务，我希望自动计算工资 | 减少人工计算错误 |
| 作为员工，我希望查看工资明细 | 了解薪资构成 |

#### 3.2.5 验收标准

```

```

#### 4.2.2 交互流程

```
用户发送指令 → 系统解析意图 → 提取关键信息 → 执行操作 → 返回结果/确认消息
```

## 📝 使用说明

### 1. 数据库迁移

执行SQL脚本创建销售人员表：

```bash
mysql -u root -p tonjclaw < database/migrations/add_salespersons_table.sql
```

或在MySQL客户端中直接执行 `add_salespersons_table.sql` 的内容。

### 2. 启动后端服务

确保Flask应用已启动：

```bash
python main.py
```

新的API端点会自动注册：
- `/api/salespersons/*`

### 3. 访问销售人员管理

启动前端开发服务器：

```bash
cd tonjclaw-web
pnpm dev
```

登录后访问：
- 销售人员管理: http://localhost:5173/desktop/salespersons
- 或通过侧边栏菜单点击"销售人员"

### 4. 使用报表筛选

访问报表页面：http://localhost:5173/desktop/reports

在日报表区域，可以使用以下筛选器：
1. 日期选择器 - 选择统计日期
2. 线路下拉框 - 选择特定线路
3. 人员下拉框 - 选择特定销售人员
4. 分类下拉框 - 选择商品分类

筛选条件会实时更新报表数据，点击"导出Excel"可导出当前筛选结果。

## ⚠️ 注意事项

### 1. 后端依赖
- 需要确保 `t_user_account` 表存在（用户表）
- 需要确保 `t_delivery_route` 表存在（线路表）
- 需要确保 `t_license_v2` 表存在（授权码表）

### 2. 权限控制
- 所有API都需要登录认证（`@login_required`）
- 销售人员数据按用户隔离（`user_id`过滤）
- 只能管理自己创建的销售人员

### 3. 数据验证
- 手机号格式验证（前端+后端）
- 线路存在性验证
- 授权码存在性验证（分配时检查）

### 4. 性能优化建议
- 销售人员数量较多时，考虑添加分页
- 报表数据量大时，考虑缓存策略
- 定期清理离职人员数据

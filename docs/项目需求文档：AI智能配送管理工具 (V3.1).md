## 1. 项目概述

### 1.1 项目背景
区县级批发商（覆盖**摩托车配件、水电五金、电动车配件、手机零配件、奶茶原料、生鲜食材、酒水饮料、日用百货**等全品类）在服务下游门店时，普遍面临订单碎片化、手工统计效率低、易出错、客户管理难等痛点。传统的人工微信接单模式已无法满足日益增长的配送效率需求。

### 1.2 核心目标
打造一款**面向区县级批发商的AI智能配送管理工具**，通过**升级桌面端软件内置AI智能体**，实现从订单接收、智能解析、自动汇总到配送报表的全流程自动化。

**技术路线**：
- **核心**：**桌面端软件全面升级**，内嵌AI智能体模块（基于大语言模型），实现高准确率的自然语言订单解析。原有**微信Hook插件**和**网站授权系统**完全保留并继续支持。
- **兼容**：用户仍可选择使用微信Hook自动捕获群聊消息，或手动录入，或未来对接小程序，所有输入均经过桌面端内置AI统一解析。
- **不变**：网站端（产品营销、用户注册、授权码购买与管理）保持不变，桌面端的授权验证机制保持不变。

## 2. 技术架构（升级说明）

### 2.1 核心变化：桌面端内置AI智能体

原有桌面端中的 `order_parser.py`（基于jieba分词+规则匹配）将被替换为 **AI智能体解析模块**。

- **技术选型**：调用本地/云端大语言模型API（如DeepSeek、通义千问等），或集成轻量级本地模型（如Ollama + Qwen）。
- **工作流程**：
  1. 桌面端收到原始消息（来自微信Hook、手动录入、或未来小程序API）。
  2. 调用AI智能体模块进行意图识别和实体抽取。
  3. 输出结构化的订单数据（客户、商品、数量、单位、备注等）。
  4. 后续订单处理逻辑（入库、汇总、报表）完全保持不变。

- **优势**：
  - 保留原有所有功能（Hook、手动、报表、授权等）。
  - AI解析准确率显著提升（尤其是口语化、缩写、同义词等）。
  - 可离线运行（若使用本地模型），保障数据隐私。

### 2.2 架构图（修订版）

```
[可选输入源]
  ├─ 微信Hook自动捕获 ──┐
  ├─ 手工录入/导入   ──┼─→ [桌面端统一消息处理]
  └─ 小程序API(未来) ──┘           │
                                   ▼
                        [AI智能体解析模块] (新增/升级)
                        - 调用大模型API / 本地模型
                        - 输出结构化订单
                                   │
                                   ▼
                        [原有订单处理引擎]
                        - 幂等写入SQLite/MySQL
                        - 商品/客户匹配
                        - 规则引擎
                                   │
                                   ▼
                        [报表/看板/导出]
                        
                        
[网站端] → 用户注册/登录 → 购买授权码 → 桌面端激活验证 (完全不变)
```

## 4. 桌面端升级详细需求（AI智能体集成）

### 4.1 功能范围
- **保留功能（100%不变）**：
  - 微信Hook消息接收服务（HTTP/TCP）
  - 所有现有API模块（订单、商品、客户、报表、授权、机器人配置等）
  - 网站授权验证逻辑（`/api/license`）
  - 本地数据库结构（不强制迁移）
  - 规则引擎（`parse_rule`表可作为AI解析的后备或辅助）

- **新增/升级功能**：
  - **AI智能体解析模块**：替换原有的`order_parser.py`核心逻辑。
  - **配置界面**：在桌面端管理后台增加AI模型配置项（选择API或本地模型、API Key、模型名称、温度参数等）。
  - **降级策略**：当AI服务不可用时，自动回退到原有的`order_parser.py`基于规则的解析，保证系统不中断。
  - **解析日志**：记录AI解析的输入、输出、置信度，便于人工复核和模型调优。

### 4.2 AI解析接口设计（内部调用）

```python
# 新模块: services/ai_parser.py
def parse_order_with_ai(raw_text: str, context: dict) -> dict:
    """
    输入: 原始消息文本, 可选的上下文(如客户历史订单、商品库)
    输出: {
        "items": [{"product_name": "土豆", "quantity": 10, "unit": "斤", "remark": "要嫩的"}],
        "customer_name": "张三",   # 可选，若能从消息中识别
        "confidence": 0.92,
        "raw_response": "..."
    }
    """
```

### 4.2.1 AI 配置与部署说明
- AI 解析模块通过环境变量控制开关，推荐在 `.env` 中配置：
  - `AI_PARSER_ENABLED=true` 启用 AI 解析。
  - `AI_PARSER_PROVIDER=custom_http` 使用自定义 HTTP API 提供商。
  - `AI_PARSER_API_URL` 填写 AI 服务接口地址。
  - `AI_PARSER_API_KEY` 根据服务需要填写 Token 或 Key。
  - `AI_PARSER_MODEL` 可选模型名称。
  - `AI_PARSER_TIMEOUT` 请求超时时间（秒）。
- 当 AI 服务不可用或解析出错时，系统会自动回退到现有的规则解析逻辑，保证业务不中断。
- 推荐生产环境使用稳定的云端 AI 提供商，测试环境可先使用本地模拟或 Mock 接口。

### 4.3 保持原有授权不变
- 授权检查仍然在订单创建前执行。
- 授权码的生成、激活、过期校验逻辑完全不改动。
- 网站端的授权管理系统继续独立运行。

## 补充内容：分行业解决方案 — 酒水配送（酒库即时配送）

### 行业画像
- **典型客户**：酒水批发商、社区酒库、烟酒行、精酿啤酒配送站。
- **下游对象**：餐厅、酒吧、KTV、便利店、烧烤摊，以及个人家庭用户（如聚会急需）。
- **业务特点**：
  - **时效性极强**：客户常要求“半小时内送到”、“冰好”。
  - **订单零碎且急**：多为临时微信/电话下单，单笔金额不大但笔数多。
  - **商品名称多样**：有品牌名（“百威”）、品类名（“精酿”）、俗称（“大乌苏”）。
  - **备注要求多**：“冰镇的”、“不要冰的”、“开瓶器要带”、“送到二楼”等。

### 痛点文案示例
> “客户在微信上喊‘来两箱百威，要冰的，快点！’——你手工记下了，转头又被另一条消息‘加一件勇闯天涯’打断。一天下来，漏送、少送、送错品牌经常发生，客户抱怨，配送员跑冤枉路。”

### AI解决方案描述
- **AI语音/文字自动识别**：即使客户发的是“百威2箱冰+勇闯天涯1件常温”，AI也能拆解为两个商品及各自备注。
- **即时订单聚合**：按客户自动合并当日/当前批次订单，生成“配送清单”。
- **紧急提醒**：对带“急”“马上”“快”等关键词的订单，系统可高亮提示或自动通知配送员。
- **冰镇/常温分离**：AI自动提取“冰镇”等要求，在报表中单独标注，避免装车时混淆。

### SEO关键词示例
- 酒水配送管理系统
- 酒库接单软件
- 啤酒批发订货系统
- 即时配送订单管理
- 烟酒行订货小程序（备选）
